Ejercicio 1) [Proyecto] Modifica el proyecto para que el servidor reciba la configuración a través de un objeto Properties. Por un lado, la clase es.uvigo.esei.dai.hybridserver.HybridServer deberá tener un constructor que reciba como único parámetro un objeto Properties y, por otro lado, la clase es.uvigo.esei.dai.hybridserver.Launcher deberá ser ejecutable y recibir como único parámetro la ruta de un fichero de configuración. De este modo, el servidor se ejecutará con: java es.uvigo.esei.dai.hybridserver.Launcher config.props

Recuerda que tanto HybridServer como Launcher deben poder trabajar con una configuración por defecto, por lo que HybridServer debería tener un constructor vacío y Launcher debe funcionar aunque no reciba la ruta de un fichero al ejecutarse.

Revisa el documento de aclaraciones del proyecto para saber cual debe ser el contenido del fichero de configuración y cual es la configuración por defecto.

Ejercicio 2) [Proyecto] Modifica el proyecto para sustituir el almacenamiento de las páginas web en un mapa por el almacenamiento en una base de datos. Cuando hagas esto, ten en cuenta lo siguiente:

  • Si el sistema está bien diseñado, la sustitución de la clase que almacenaba las páginas en un mapa por la nueva clase que las almacenará en base de datos no debería suponer ningún cambio en ninguna otra clase de la base de datos, salvo en aquella que crea las instancias de esta clase. Esto es lo mismo que lo visto en el Dojo 3.
  • La clase que almacena las páginas web en la base de datos no debería tener ninguna dependencia con una base de datos concreta. Es decir, el hecho de que utilice JavaDB, MySQL u otro SGBD no debería suponer ninguna modificación en dicha clase. Para ello, lo más sencillo es que esta clase reciba el objeto Connection en el constructor o a través de un setter.

Ejercicio 3) Crea una aplicación que te permita gestionar una base de datos de personas. Al menos, debería permitir realizar altas, bajas y listar las personas de la base de datos. Para ello parte de cero (no reutilices ningún código del utilizado en los Dojos) e intenta estructurar la aplicación lo mejor posible. Es importante que crees una clase que te permita representar la entidad Persona (del mismo modo que Employee representaba a la entidad empleado en el Dojo 3).

Ejercicio 4) Añade a la aplicación creada en el ejercicio 3 la capacidad de almacenar las mascotas de cada persona. Ten en cuenta que para ello deberás crear la entidad Mascota y modificar la entidad Persona para que contenga las Mascotas de las que es propietario. Para ello, por ejemplo, la entidad Persona puede tener el método "getMascotas():Set<Mascota>". Debes pensar muy bien como implementar los objetos DAO para que esto sea posible.

Ejercicio 4) Extiende la aplicación de los ejercicios 2 y 3 para que Mascota también tenga un método "getAmo():Persona".

Ejercicio 5) Abre el proyecto Transactions en Eclipse y evalúa el efecto de los distintos tipos de aislamiento (isolation levels) en los problemas de las transacciones. Para ello vete a la clase TransactionExamples y ejecútala modificando los valores de la constante ISOLATION_LEVEL. Completa una tabla como la vista en la página 44 del tema 4 utilizando JavaDB y utilizando MySQL.

Ten en cuenta que es necesario incluir los drivers de la base de datos que vayas a utilizar.
